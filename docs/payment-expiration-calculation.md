# Payment Expiration Time Calculation - Order System

## 1. Quick Answer

**Thời gian hết hạn thanh toán được tính như nào?**

```
Expiration Time = Order Created Time + 900 seconds (15 minutes)

Formula:
- Order CreatedAt: T0 (timestamp khi payment được tạo)
- Current Time: Tnow
- Time Passed: Tnow - T0
- Remaining Time: 900 - (Tnow - T0)
- Expired when: Remaining Time <= 0
```

**Cấu hình:**
- **Duration**: 900 giây (15 phút) - có thể thay đổi via env var
- **Environment Variable**: `VITE_ORDER_EXPIRATION_TIME_SECONDS=900`
- **Update Interval**: Cập nhật mỗi 1 giây (countdown)
- **Network Buffer**: Trừ thêm 30 giây để đảm bảo có đủ thời gian xử lý trước khi hết hạn

## 4. Frontend - Countdown Timer Calculation

### 4.1 OrderCountdown Component

**File**: `order-ui/src/components/app/countdown/OrderCountdown.tsx` (Lines 25-49)

```typescript
const OrderCountdown = ({ createdAt }: { createdAt: Date }) => {
  const [remainingTime, setRemainingTime] = useState(0)
  const [isExpired, setIsExpired] = useState(false)

  useEffect(() => {
    const timerInterval = setInterval(() => {
      const now = moment()
      const orderCreated = moment(createdAt)

      // Tính thời gian đã trôi qua (giây)
      const timePassed = now.diff(orderCreated, 'seconds')

      // Tính thời gian còn lại = 900 - timePassed
      const remaining = 900 - timePassed

      if (remaining <= 0) {
        setIsExpired(true)
        clearStore()           // Xóa payment store
        clearInterval(timerInterval)
      } else {
        setRemainingTime(remaining)
      }
    }, 1000)  // ← Update mỗi 1 giây

    return () => clearInterval(timerInterval)
  }, [createdAt])

  // Hiển thị MM:SS format
  const minutes = Math.floor(remainingTime / 60)
  const seconds = remainingTime % 60

  return <div>{minutes}:{String(seconds).padStart(2, '0')}</div>
}
```

**Công thức tính:**
```
Time Passed (seconds) = now.diff(createdAt, 'seconds')
Remaining Time = 900 - Time Passed

Ví dụ:
- createdAt: 14:00:00
- now: 14:05:30
- Time Passed: 330 seconds (5 phút 30 giây)
- Remaining: 900 - 330 = 570 seconds (9 phút 30 giây)

Khi Remaining <= 0:
  → Order hết hạn
  → setIsExpired(true)
  → Dừng polling
  → Xóa payment store
```

export const CustomerDisplayCountdown = ({ createdAt }: Props) => {
  const [timeRemaining, setTimeRemaining] = useState(900)

  useEffect(() => {
    const interval = setInterval(() => {
      const now = moment()
      const created = moment(createdAt)
      const elapsed = now.diff(created, 'seconds')
      const remaining = 900 - elapsed

      if (remaining <= 0) {
        setTimeRemaining(0)
        // Trigger: Order expired
      } else {
        setTimeRemaining(remaining)
      }
    }, 1000)

    return () => clearInterval(interval)
  }, [createdAt])

  return (
    <div className="fixed top-0 left-0 right-0">
      <span>Hết hạn trong {formatTime(timeRemaining)}</span>
    </div>
  )
}

const startPolling = () => {
  pollingInterval = setInterval(async () => {
    try {
      // Lấy thông tin order mới từ server
      const updatedOrder = await refetchOrder()

      // Kiểm tra nếu đã thanh toán
      if (updatedOrder.orderStatus === OrderStatus.PAID) {
        // Hủy polling
        clearInterval(pollingInterval)
        // Chuyển hướng tới trang success
        navigate(`/client/order-success/${updatedOrder.slug}`)
      }
    } catch (error) {
      console.error('Polling error:', error)
    }
  }, 2000)  // ← Polling mỗi 2 giây
}

## 6. Gift Card Polling

### 6.1 Gift Card Polling with Expiration Check

**File**: `order-ui/src/hooks/use-gift-card-polling.ts`

```typescript
export const useGiftCardPolling = (
  orderId: string,
  expirationTimeInSeconds: number,
  onExpired?: () => void
) => {
  const [isPolling, setIsPolling] = useState(false)

  useEffect(() => {
    if (!isPolling) return

    const polling = setInterval(async () => {
      try {
        // Fetch order status
        const order = await fetchOrder(orderId)

        // Tính expiry time
        const orderDate = new Date(order.createdAt)
        const expiryDate = new Date(
          orderDate.getTime() + expirationTimeInSeconds * 1000
        )
        const now = new Date()

        // Kiểm tra hết hạn
        if (now > expiryDate) {
          setIsPolling(false)
          onExpired?.()  // Gọi callback khi hết hạn
          return
        }

        // Kiểm tra payment completed
        if (order.paymentStatus === 'completed') {
          setIsPolling(false)
          onSuccess?.()
          return
        }
      } catch (error) {
        console.error('Polling error:', error)
      }
    }, 30000)  // ← Polling mỗi 30 giây

    return () => clearInterval(polling)
  }, [isPolling])

  return { isPolling, setIsPolling }
}
```

## 7. Payment Cancellation

### 7.1 Automatic Cancellation on Timeout

Khi order **hết hạn** (900 giây):

```
OrderCountdown: 00:00
    ↓
setIsExpired(true)
    ↓
clearStore()  (xóa payment method)
    ↓
Dừng polling
    ↓
Hiển thị: "Đơn hàng đã hết hạn"
```

### 7.2 Manual Cancellation API

**File**: `order-api/src/payment/payment.utils.ts` (Lines 40-76)

```typescript
async cancelPayment(paymentId: string) {
  const payment = await this.paymentRepository.findOne({
    where: { id: paymentId }
  })

  // Cập nhật payment status
  payment.statusCode = PaymentStatus.CANCELLED
  payment.statusMessage = 'Thanh toan da bi huy'
  payment.deletedAt = new Date()

  await this.paymentRepository.save(payment)

  // Hủy QR code trên bank nếu cấu hình cho phép
  if (this.configService.get('MODE_CANCEL_QR_BANK_TRANSFER')) {
    await this.bankTransferStrategy.cancelQRCode(payment.qrCode)
  }

  return payment
}
```

## 9. Complete Flow Diagram

### 9.1 Payment Initiation to Expiration

```
Step 1: User clicks "Thanh toán"
    ↓
Step 2: POST /payment/initiate
    ├─ Backend: Create Payment entity
    │  └─ createdAt: Timestamp hiện tại (e.g., 14:00:00)
    └─ Response: { qrCode, amount, createdAt }

    ↓

Step 3: Frontend: OrderCountdown component mounts
    ├─ Receive: createdAt = 14:00:00
    └─ Start interval: update mỗi 1 giây

    ↓ (Mỗi 1 giây)

Step 4: Calculate remaining time
    ├─ now = current time
    ├─ timePassed = now - createdAt
    ├─ remaining = 900 - timePassed
    ├─ Update UI: MM:SS format
    └─ If remaining <= 0:
       ├─ setIsExpired(true)
       ├─ clearStore()
       ├─ clearInterval()
       └─ Show: "Đơn hàng đã hết hạn"

    ↓

Step 5: During countdown, polling happens
    ├─ Interval: 2 seconds
    ├─ GET /orders/{slug}
    └─ Check: orderStatus === PAID?
       ├─ YES: Stop polling, navigate to success
       └─ NO: Continue polling

    ↓

Step 6: Two outcomes

    Outcome A: User completes payment BEFORE 15 min
    ├─ Bank receives payment
    ├─ orderStatus changes to PAID
    ├─ Polling detects PAID
    └─ Navigate to success page

    Outcome B: Timer reaches 00:00 WITHOUT payment
    ├─ isExpired = true
    ├─ Polling stops
    ├─ Payment store cleared
    └─ User sees "Order expired" message
```

### 9.2 Countdown Update Timing

```
14:00:00 - createdAt
14:00:01 - remaining: 899s (14:59)
14:00:02 - remaining: 898s (14:58)
...
14:14:59 - remaining: 1s (00:01)
14:15:00 - remaining: 0s (00:00)
          - setIsExpired(true)
          - Stop countdown
          - Stop polling
```

## 11. Key Technical Points

### 11.1 Why 900 Seconds?

```
900 seconds = 15 minutes

Reasons:
1. Time for customer to scan and complete payment via bank app
2. Time for bank to process transfer (usually < 5 minutes)
3. Buffer for network delays
4. User-friendly duration (not too short, not too long)
```

### 11.2 Update Interval: 1 Second

```
Why 1 second instead of 5 or 10?

Benefits:
✅ Real-time visual feedback for user
✅ Precise countdown display
✅ Quick response to isExpired state

Drawbacks:
❌ Slightly more CPU usage
❌ More function calls

Compromise: 1 second is acceptable for frontend countdown
```

### 11.3 Network Buffer: 30 Seconds

```
Why 30 seconds?

Scenario 1: Normal network
├─ User clicks Pay button
├─ Network latency: 50ms
└─ Completes instantly

Scenario 2: Slow network
├─ User clicks Pay button
├─ Network latency: 10+ seconds
└─ Takes longer to process

Buffer of 30 seconds ensures:
✅ Even on slow networks, request goes through
✅ Countdown shows accurate remaining time
✅ Prevents timing-out before server processes
```

### 11.4 Polling Interval: 2 Seconds

```
Why 2 seconds?

Polling every 2 seconds means:
- 30 requests per minute (reasonable)
- 1800 requests per 15 minutes (max for 1 order)
- Quick response when payment completes

If interval were too long (e.g., 30s):
❌ User waits 30 seconds to see success page
❌ Poor UX

If interval were too short (e.g., 500ms):
❌ Server load too high
❌ Network congestion
```
